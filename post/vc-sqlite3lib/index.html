生成SQLite静态库

debug版和release版
Mar 18, 2022

使用vs2019(v16.11)

.\vs_Community.exe --layout D:\win10soft\vs2019 `
--add Microsoft.VisualStudio.Component.CoreEditor `
--add Microsoft.VisualStudio.Workload.CoreEditor `
--add Microsoft.VisualStudio.Component.Roslyn.Compiler `
--add Microsoft.Component.MSBuild `
--add Microsoft.VisualStudio.Component.TextTemplating `
--add Microsoft.VisualStudio.Component.VC.CoreIde `
--add Microsoft.VisualStudio.Component.VC.Tools.x86.x64 `
--add Microsoft.VisualStudio.Component.Windows10SDK.19041 `
--add Microsoft.VisualStudio.Component.VC.Redist.14.Latest `
--add Microsoft.VisualStudio.ComponentGroup.NativeDesktop.Core `
--add Microsoft.VisualStudio.Workload.NativeDesktop --lang en-US

Debug版

新建DLL工程

project name: sqlite3（使用这个名字主要是在生成lib的时候不用更改输出名了）

添加SQLite源文件

下载sqlite-amalgamation-3380100.zip， 把sqlite3.h、sqlite3ext.h和sqlite3.c拷贝到工程根目录。

编译工程

预编译头文件

不使用预编译头文件。

Project ==> Properties ==> C/C++ ==> Precompiled Header

如果不更改会出现如下错误：

sqlite3.c : fatal error C1853: 'Debug\sqlite3.pch' precompiled header file is from a previous version of the compiler,
or the precompiled header is C++ and you are using it from C (or vice versa)

生成文件类型

Project ==> Properties ==> General

编译即可。

Release版

從Precompiled Binaries for Windows下载sqlite-dll-*.zip

output *.lib

Start menu -> All Programs -> Visual Studio 2019 -> x86 Native Tools Command Prompt for VS 2019 Enter the following command:

lib /def:sqlite3.def /out:sqlite3.lib /machine:x86

  • Creating a Windows .lib file for SQLite for use with Visual Studio
